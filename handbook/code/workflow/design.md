# Design

## Goal

1. whether the code is large to handle correctly. if not, the module should be much more smaller.
2. whether the spec is clear and concise enough to guide the implementation.

## Steps

1. write `spec` in `docs/spec`
2. write decision in `docs/dev/decision`
2. write plan in `docs/dev/plan`with same file name.
3. write a example python module first, like `examples/work/journal.py`
4. then write a report on what is the different with the code and plan, 
so that I can know your innovation, on `docs/dev/report`.
5. Then evaluate the code with the spec, decision, the plan and the report on `docs/dev/evaluation`
6. write retrospective on `docs/dev/retrospective/`
