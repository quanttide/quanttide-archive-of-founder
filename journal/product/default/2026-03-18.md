## 量潮数据

基本流程：
1. 澄清：客户描述需求，我们评估需求，达成共识或者达不成共识。
2. 设计：设计方案，给客户确认各种细节，达成共识。
3. 实施：执行方案，报告阶段性成果，达成共识。
4. 交付：我们交付，客户评估，达成共识。
5. 复盘：我们提出问题，客户给建议，达成共识。

### 决策

假设不做账号系统，而是靠人工维护数据进入系统。

我们是应该从全流程打通呢，还是应该从单个流程开始出发？全流程才是核心价值，充分利用 Vibe Coding 的能力大踏步前进后退。

那么它可以是管理后台的数据业务模块。我们可以尝试把这个模块快速写个 Flutter 客户端上线看看。数据可以先存 GitHub 的私有仓库 qtdata-profile。

如何从需求出发设计页面和组件？
先聚焦于澄清和交付两个最小环节。

有没有可能从档案开始建起，然后在档案上搭客户端。这样可以和 Flutter 客户端同步，用来备份业务数据。
