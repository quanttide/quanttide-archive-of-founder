# 2026-03-31 思考云

## 产品定位

交付：

1. 一个想法的原子定义
   什么可以放进容器？最小的、不可再分的“想法单元”应该包含哪些本质要素？例如：身份、时间、内容、来源、状态。
2. 容器之间的连接规则
   想法不是孤立的。容器与容器之间允许怎样的关系？父子、引用、矛盾、演化？本质是关系类型的最小集合。
3. 容器的生命周期模型
   一个想法从进入容器到离开容器会经历哪些本质阶段？例如：原始捕捉 → 待处理 → 已归档 / 已废弃 / 已转化为行动。每个阶段的意义是什么。
4. 约束边界
   容器不能做什么？哪些操作是被禁止的（如自动删除、未经同意的合并）？本质是用户主权与自动化之间的红线。
5. 成功标准
   什么情况下说这个容器是“有效的”？本质指标：降低想法丢失率、减少重复思考成本、提升回溯速度，而不是功能数量。

功能：

1. 捕捉
   在任何时刻、以最低摩擦接收想法，不评判、不丢失。
2. 存储
   持久化保存想法及其元数据，确保可回溯、可审计。
3. 关联
   自动或半自动地发现想法之间的潜在连接，帮助用户看到自己思维的图谱。
4. 约束执行
   确保用户设定的边界（如不自动删除、债务显式化）被系统强制执行，不被违反。
5. 沉淀
   将零散想法中具有长期价值的部分，自然地迁移到更有序、可复用的形态（如知识库、行动项、债务卡片）。
6. 遗忘
   提供明确的机制让用户安全地丢弃不再有价值的想法，避免容器无限膨胀。
