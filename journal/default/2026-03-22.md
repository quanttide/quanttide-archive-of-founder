## 第二组

我想象了产品形态。其实核心是一个批流一体的数据状态机，支持create和update。思考的时候，是两个AI轮流说，我批注或者回答，有点类似于那种对话游戏。Opencode也有这个机制，不过可以做的更游戏化一些。写作的时候要围绕原文改。

孤独感让我希望被AI看见，所以我希望让AI看到我的思考或者写作轨迹，从而激发我的动力。

我尝试把待办的完整想法都记录在待办空间里，但是还是觉得有点不放心。我又记录到了系统提醒里。刚想起来还缺了一件事。

工作档案里似乎应该放写作计划、叙事思路等等叙事工程的信息，但是还不知道放什么是有效的。

执行管理这个职能就是一个跨期调控工具，可以让执行状态和执行需求不匹配的时候暂存。而难点其实就在于怎么和自己达成有效契约。因为稍微不管就很容易又乱了。基本思想是执行区只有一小部分在做，其他的大部分都被系统管理着。如果形成了连续的效果，那么系统会自动提醒重复，或者帮忙调整合适的事情拿出来，或者合并等。

我最近在反复思考的一个主题是“建立说一不二的王权”，通过这种影响力和市场的不合理对抗，在市场上打赢来获得增长。

怪不得我最近沉迷电脑维护第二大脑。这个就是养龙虾的后续。现在看他自己工作那种快乐感压过了所有烦躁，难怪我喜欢这个。就好像没有在工作又在创造产出一样，满足了我很久以来的一个困惑。

### 第三组

我刚才想了想，决定还是先不急着把个人命令改公司命令。我觉得还是场景太不清楚。个人命令甚至不需要考虑命名空间，反正自己能记得，用着用着直觉应该就建立起来了。把我自己放到擅长的用户位置上，这才是真正的氛围编程。

我也在改变我自己的习惯。以前习惯要把一切都设计好。这种开发习惯越来越不适合当下了。还是这种靠用户反馈迭代的方式更适合我。也不要怕版本号多。这种迭代就是要这样高速，版本号起来了才对。也不用担心命名空间的问题，很快迭代起来用起来就会知道了。

这种安心感和刚才外面散步的时候那种慌乱完全不一样，从文字的感觉里也比较镇定。感觉这个系统和我自己的和谐程度在提升，我开始越来越缩紧了窗口让 AI 自己设计思路。我觉得这越来越符合我的迭代习惯。

我在探究的元问题是，如果我们必须要靠小步快跑来逐步让流程简化，而这个过程中 Vibe Coding很容易崩溃，我们要怎么系统性地阻止这种崩溃呢。我们现在这套雪花编程法就是关键。

我在思考的这些问题，没有人会给我答案。我觉得我们需要慢慢探究，首先要提出这些问题，给出我们的答案并且持续观察。如果确实能够根本性地影响到业务，那就是顺利的。

我觉得虽然现在还只能在限定的范围内做事，但是这种能力很快会越来越清楚的。我觉得会有很大的变化发生。

我们现在是在封装 GitHub 和 git，下一步也许是 terraform 和各种基础设施，再下一步或许是各种细节。我觉得我关注的概念定义等细节问题慢慢地浮现出来，这些都是我平时爱思考和研究的。比如说目前的 git 状态机。

### 第四组

我问DeepSeek，我做的工具有什么意义。DeepSeek说原生工具只提供接口不提供工作流和最佳实践，这是为了让上层可以自由组合。所以我要定义的本质上是我们自己的最佳实践为中心的体系。比如我最近时常被这些submodule困扰，那么就可以慢慢定义清楚，把这些问题记录下来、封装起来。

教程的原生操作和手册的理想操作之间的距离就是工具要磨平的。难点主要还是在写出来流程，开发的难点倒是其次。可以通过我这样的流程来优化。所以真正的工作难点就在观察和记录经验了。那么这个就可以充分利用大模型聊天记录来审计了。或许这就是一个重要的审计工具呢，甚至是智能体研发工具呢。

新需求是层出不穷的。但是以目前的工程能力还做不到每个都试一遍，所以只能先做想法清晰一些的。

我在考虑在这个模块成熟以后直接往库里封装，这样可以直接上生产改 GitHub actions 和 pre-commit。我觉得这是一个不错的思路或许。其实一个核心思想就是封装最佳实践，减少手动重复操作。

关机又开始焦虑和怀疑做的需求有什么意义，主要是对团队的。感觉还是在打转，没有真正深入团队需求来。不过这个事情就是不好做。我又在想，那么既然已经定义了问题，那么就存在最佳实践。我要做的事情就是不断总结最佳实践，然后封装，然后压缩最佳实践的密度到手册里。或许这就是必要的路径。

我要研发产品就需要具体地描述我对产品的想象，否则是很难把这些东西落地的。或许这就是可以练习二次重写的契机。他和写作文不一样，更多需要的是理性，这很适合被 AI 复制。而且我写的也不是标准 PRD，更多是产品蓝图。我觉得这是代替 PRD 的一种好办法。其实是在写各种用户场景。

比如说，我现在想象的就是，未来工作可以高度自动化，每次想一堆以后就可以让 AI 排着队干活。这样就可以让 AI 帮我们把研发都干了，人就可以充分发挥创造力。

也就是产品档案每个产品的 index 作为产品愿景介绍清楚，剩下的让 AI 接管设计就好了。或许可以试一试这些新方法。

我今天好像突然打通了任督二脉一样，突然就知道了思考、执行、产品如何才能有效发挥职能。但是这是怎么发生的呢？和思考过程有关还是思考结果有关还是都有关？我怎么突然一下子就找到了可以让工作发生的方法呢。
