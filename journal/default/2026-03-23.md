# 2026-03-23

## 第一组

Opencode改变了我的研发节奏。现在的重点在于写出准确的反馈给AI修改。我觉得我又需要调整自己的习惯了。

比如说，我发现 AI 写的 cli刚好戳中了我的需求，但是又不能够直接让我觉得可用，我就去修改。然后一个比较容易纠结的问题是领域划分，这个会影响 cli 的整洁度。也不知道接下来会往什么方向演化。我已经初步想好了，就让 AI 给重写一遍，看看这个办法能不能更好地处理。

我在不断尝试砍掉需求，再不断地删除。我觉得这样折腾我自己的外脑项目很好玩。反正也是自己玩的，不需要什么规范。这样也就是说，真正的产品平台需要更多这样的实验室，来让产品可以被快速验证。

我在尝试模拟新人的认知。这种对不齐的感受确实是挺不好受的，也是要逐渐加强的点。我在考虑如何有效加强。或许这需要审计才可以。

想了想，可能需要 Opencode 的原始对话。这是一个可以私有化的重要资产。或许还是要考虑把个人私有档案恢复起来存各种需要备份的数据。就是可以改个名字就是了。

这或许不是坏事。当我开始需要有私有信息存起来的时候，就说明我有大量的实践开始进步了。虽然暂时还无法确定是哪种进步。

突然想到，比如一直想做的 Web3 机制或许可以做起来。首先就可以从奖励自己和 AI 开始做起。这样或许可以提供良好的反馈数据来证明 AI 做出了什么贡献，或者人机交互做出了什么贡献。不过这样 AI 的身份认证就成了难题。

我开始对未来的开发乐观起来，我感觉到我的想法越来越找到了正确的表达方式，我的表达力在变得更顺畅。这个系统一旦搭起来，它就可以自主生长了。团队也可以在这个系统上继续做事。我的视野足够给这个系统注入许多完全不同的活力，能够让这个系统运转地不失标准化又不失弹性。

## 第二组

我一直在等待或者寻找一个状态，就是什么时候我可以把我的个人大脑和公司大脑对接起来，以及什么时候能把GitHub和飞书对接起来。前者不难，把整个量潮第二大脑作为元认知数据库即可。后者感觉需要一些合适的契机或者不得不做的事情。

Kimi说保留其中思考的转折很有价值。我这样听说以后，想到的是要把档案按照时间线这样一点点备份下来，这样就可以留给未来更多历史资料研究其中的演化过程了。

突然想到，我给第二大脑留下的大量空白，或许未来可以通过系统去填充，而不是我现在这样缓慢的手动积累。所以留一大堆空白没错，恰恰是我已经看到了系统高速演化可能经历的过程并且提前准备好路标。

就是担心自己到时候能不能认得。从现在的实践看，很多时候是能认得但是隐隐约约觉得有点碍事。不过相信 AI 可以帮助我解决这个问题。

## 第三组

Kimi 建议说不需要完全认得。我觉得有意思。这类我已经有想法的思考，看起来不太需要 Kimi 和 DeepSeek 帮助我，更多时候是手动挡的总结记录。

Kimi说需要一些飞书和 GitHub 连接工作的契机。看起来两个系统之间的咬合度不高，可能是刻意这样的。我感觉我这种产品思维和以往都不一样，以往是非常按部就班地一个一个单元设计，非常有拼图思维。现在的产品思维是流动和生长的，像雪花一样展开，可以叫它雪花产品法，是雪花写作法的延伸。

我发现创意写作里的很多方法都可以成为元方法。我想这是为什么创意写作打开了我的思路的原因。或者说，创意写作的工作流具备很强的元工作流的性质，是可以加强 CODE 循环的方法。或者说，给 CODE 循环明确了迭代节奏，也给迭代的阶段性成果明确了样貌，很符合敏捷开发。这些方法之间开始联系起来影响了整个元认知系统。这种感觉很奇妙。我开始更加确认其中的关联，并且找到了显性化的入口。

我有点想到 infra 的模拟方法了，用来部署服务端。这样可以在后台用 k8s 管一个相对稳定的客户端。有了 AI 之后，我也不怕设计 k8s 的复杂了，也放弃了卡手的 FaaS 部署应用。FaaS 留下来跑应用层就可以，平台层还是需要 k8s 作为基础。

我感觉整个系统的样貌又清晰了一些。我去更新产品日志。更新完了，感觉又清晰了一些。这种感觉真的很奇妙。我感觉到自己的创造力在快速增长。我开始能够想象到自己给自己搭的路标在哪里，要长成什么样子了。

我第一次感觉到我的想法具备了完全落地的可能性，我的外延被大大扩展了。而且未来团队也可以复制这种状态，我们积累的这个完整的外脑会更加地好用快速生长，最终长成参天大树。最可贵的一点将是它适应环境不断加速的能力。

### 第四组

我现在才感受到这些会不会太晚。我觉得不会。我想要造的系统是一个自然生长的复杂系统，这不是简单的接个项目的事情，而是要跟着我们自己的进化而进化的系统。要跟得上就得保证绝对安全的执行力。

我想要做个日记应用但是自己的数据不想让团队看到，这就涉及到复杂的混合云架构，要明确哪些权限给团队、哪些自己留着，感觉要在后台留大量的后门了。

这些想法或许都是未来系统进化的动力。只是我也不确定什么时候这些想法才可以转化为生产力。我觉得我好像有一种很机械的想法，就是积累数据给 AI 读。但是因为我也在同步思考，AI 给我反馈以后我也在给新的反馈，所以数据更多的作用是围绕某个点把前后串起来看到更深的规律。虽然也不知道什么时候会发生，但是这种认知让我安心，既不觉得太上头又不觉得太无用。

这种创造过程是曲折，所以它就非常需要能够有更准确的反馈，让人机可以围绕更多的原型来交流，而不是直接面对 AI 可能产出的混乱。
