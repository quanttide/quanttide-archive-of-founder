## Beancount

存储可以选择对象存储或者GitHub。GitHub一定要加密，理论上对象存储可以用内置的加密。也要看 Beancount 社区现在的支持是什么样子。
GitHub 目前有 quanttide 作为开源，quanttide-archive 归档，quanttide-tech 作为内部日常协作。账本比较敏感，放在哪里似乎都不太合适，再开一个 org 又比较增加管理成本。
但是只放在对象存储也不太安全，还是需要一个额外的备份方案，比如都用坚果云加密备份。
git-crypt 可以加密 git 仓库，或许是一个好办法。把坚果云作为加密备份。
所以重点还是先用 git 加密把账本建立起来再考虑下一步。
