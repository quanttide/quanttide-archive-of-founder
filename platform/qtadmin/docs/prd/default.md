# 默认活动：增量式被动观察

## 1. 产品概述

### 背景

用户在写工作日志时，希望 AI 能自动为内容添加结构化批注，但不希望 AI 重写原文。

### 目标

实现"嵌入式批注"功能：AI 不负责全篇重写，只负责"扫描"和"打标"，在用户写作停顿时自动为段落添加观察者视角的结构化信息。

## 2. 用户故事

作为写作者，我希望 AI 在我完成写作后，通过命令行为段落添加结构化批注，但不改我的原文，以便我能保留完整的思维过程同时获得 AI 的洞察。

## 3. 功能需求

### 3.1 段落切分

- 按 `\n\n`（空行）将全文切分为多个"逻辑段落"
- 过滤：忽略只有标点、代码块或过短（<20字）的段落

### 3.2 状态检测

- 检查每个段落是否已存在 `🤖 观察者注` 标记（不加粗体，与 Prompt 输出保持一致）
- 已存在 → 跳过（保持幂等性）
- 不存在 → 标记为"待观察段落"

### 3.3 AI 提炼

针对每个待观察段落，AI 进行轻量级推理，判断核心价值：
- 若无意义闲聊 → 输出 SKIP
- 若包含洞察/决策/风险 → 按格式输出批注

### 3.4 合成回写

将原文段落 + AI 生成的批注拼接，写回文件

## 4. 验收标准

### 4.1 功能验收

- [ ] 输入 Markdown 文件，输出带批注的 Markdown 文件
- [ ] 原文完全保留，不做任何修改
- [ ] 已处理段落不重复处理（幂等性）
- [ ] 短段落（<20字）不处理

### 4.2 格式验收

批注使用 Markdown 引用格式（`>` 前缀），结构如下：

```
> 🤖 观察者注
> 🏷️ 标签：#分类1 #分类2
> 💎 提炼：提取的标准化知识
> ⚠️ 状态：状态说明（可选）
> 🔗 关联：关联说明（可选）
> 🔑 关键：关键要点（可选）
> 🔄 模式：模式说明（可选）
```

### 4.3 性能验收

- 单次请求处理时间 < 30 秒
- Token 消耗 < 5000/次

## 5. 非功能需求

### 5.1 触发方式

手动触发：`python -m thera.cli <file-path>`

### 5.2 错误处理

- 网络异常：重试 3 次后返回错误
- 解析异常：跳过该段落，记录日志

### 5.3 成本控制

使用 opencode 免费模型
