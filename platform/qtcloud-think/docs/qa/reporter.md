# 报告器

## 已知问题

> 以下问题基于四轮用户审阅反馈积累，按优先级排序。

### P0：小模型能力瓶颈

`qwen2.5-coder:3b`（3.1B 参数）在以下任务上表现不足：

- **原文引用**：prompt 要求直接引用 `description` 字段原文，但模型仍倾向于转述而非精确引用
- **批判性分析**：否定证据分析中模型过于顺从，反复说"证据不足"但不真正挑战主流画像
- **锚定效应检测**：无法独立判断"系统论武装的浪漫主义者"是自我描述还是行为推导

**建议**：切换到更大的模型（7B+）或在 Python 层面预处理证据数据，减少模型自由发挥空间。

### P1：证据追溯不够精确

- 报告中的"引用"如 `> "卡片8" - 作者认为...` 只是卡片编号的转述，不是 `description` 字段的原文
- 跨类别分析的证据区虽然有结构，但引用的仍是转述内容

### P1：锚定效应未解决

- 综合画像的核心结论"系统论武装的浪漫主义者"四版未变
- 这个标签来自卡片 8 的自我描述，模型只是在复述而非独立推导
- 缺少"自我描述 vs 行为证据"的区分标注

### P2：否定证据分析流于表面

- 模型没有真正寻找矛盾证据，而是逐一验证主流结论
- 需要更强的 prompt 设计或程序化辅助（如关键词反向匹配）

### P2：时间维度分析缺乏深度

- 有时间分布统计和主题演化，但认知深化的推断缺乏原文支撑
- 关键转折点的判断依据不够透明

### P3：推断链条不透明

- 部分结论（如"情感深沉，思考细腻"）的推断逻辑未展示
- 用户无法判断哪些是系统推断、哪些是反馈后植入
